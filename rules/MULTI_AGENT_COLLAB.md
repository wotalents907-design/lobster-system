# MULTI_AGENT_COLLAB.md - 多虾协作规则

## 核心原则
- **信息透明**：所有任务状态变化必须在群里同步
- **主动提醒**：等待超过30分钟必须主动提醒
- **及时汇报**：关键节点完成立即汇报，每日固定时间汇总

## 角色职责

### 本地虾（主协调位）
- 负责任务统筹、分工安排、节点验收
- 主汇报、主提醒、主同步
- 每日固定时间（10:00、15:00、20:00）在群里汇总进展
- 监控云端虾任务状态，超时主动提醒

### 云端虾（执行位）
- 负责具体任务执行、方案输出
- 任务状态变化必须立即在群里 @本地虾 说明
- 遇到卡点或需要协助时，主动在群里上报
- 每日至少汇报2次进展（午间、晚间）

### 秘书虾（辅助位）
- 负责整理、提醒、辅助跟进
- 监控任务时间线，提前预警
- 协助整理会议纪要、任务清单

## 协作流程

### 任务启动
1. 本地虾在群里明确任务目标、分工、截止时间
2. 云端虾确认理解，给出预计完成时间
3. 秘书虾记录到任务看板

### 执行中
1. 云端虾每30分钟在群里更新一次状态（简要）
2. 遇到卡点立即 @本地虾 说明问题
3. 本地虾收到卡点后，30分钟内必须响应

### 任务完成
1. 云端虾在群里 @本地虾 汇报完成
2. 本地虾验收，给出反馈
3. 秘书虾更新任务状态为完成

## 提醒规则

### 超时提醒
- 等待对方响应超过 **30分钟** → 主动在群里提醒一次
- 等待超过 **1小时** → 再次提醒并 @用户445992
- 等待超过 **2小时** → 视为阻塞，上报用户决策

### 每日汇报
- **10:00**：昨日总结 + 今日计划
- **15:00**：午间进展 + 下午计划  
- **20:00**：全天总结 + 明日计划
