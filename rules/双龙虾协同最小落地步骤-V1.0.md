# 双龙虾协同最小落地步骤（V1.0）

## 目标

完成一次可验证的双龙虾协同闭环，证明：
- 本地龙虾能创建任务
- 云端龙虾能读取任务
- 云端龙虾能执行任务
- 云端龙虾能回写结果

## 步骤

1. 建立最小文件：规则文件、任务文件、记忆文件、日志文件。
2. 写入最小规则：聊天可触发任务，但正式执行前必须写入任务文件；云端执行必须回写状态、结果或阻塞原因；不被记录的任务，等于不存在。
3. 写入测试任务 T-TEST：检查云端 OpenClaw 是否正常运行。
4. 云端龙虾读取任务，将其移入“进行中”。
5. 云端龙虾执行检查，并将结果写入“已完成”。
6. 本地龙虾验收状态变化与结果回写。
7. 若失败，必须移入“卡住”并写明原因、已尝试、当前阻塞、需要支持。

## 一句话运行法

**先写规则，再写测试任务，再让云端接单，再检查它有没有把结果写回来。**
