# 双龙虾协同系统 V2.1（可上线版）

## 系统目标

实现本地龙虾总调度、云端龙虾执行与云端内调度、子代理细分执行，并满足：任务可记录、状态可见、结果可追踪、双端可同步、完成可通知、失败可重试。

## 角色定义

### 本地龙虾（总调度脑）
- 接收用户任务
- 拆解任务
- 写入任务系统
- 维护规则
- 决定任务优先级
- 汇总执行结果

### 云端龙虾（执行主脑）
- 读取任务系统
- 接收任务
- 在云端执行任务
- 在云端内部调度子代理
- 回写状态、结果、阻塞信息
- 发送任务完成/卡住通知

### 子代理（细分执行单元）
- 接收云端龙虾分配的子任务
- 只执行，不做总决策
- 回传结果、错误、阻塞信息

## 同步机制

采用 Git 同步。

### 单写方原则
- 任务目标、任务创建、任务优先级：本地龙虾写
- 执行状态、执行结果、卡住信息：云端龙虾写

## 标准执行流程

1. 本地创建任务并提交同步。
2. 云端自动拉取并接单。
3. 云端执行并持续更新状态。
4. 完成或卡住后回写并推送。
5. 飞书发送完成/卡住通知。

## 飞书通知规则

### 必须通知
- 任务完成
- 任务卡住

### 可选通知
- 心跳通知（必须限频）

## 失败处理与重试规则

- Git pull 失败：记录失败时间、错误信息、下次重试时间
- Git push 冲突：暂停自动覆盖，记录冲突原因，进入卡住或等待重试
- 飞书通知失败：记录到日志，不得静默吞掉
- 子代理超时：记录超时，回收任务，重新分配或转卡住

## 一句话总结

**V2.1 不是两个龙虾聊天，而是本地定任务、云端执行、Git 同步状态、飞书回传结果的可上线协同系统。**
