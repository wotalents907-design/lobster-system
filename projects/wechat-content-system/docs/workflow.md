# 公众号双号内容系统工作流

## 一、内容生命周期

1. 想法进入
2. 判断归属（客户号 / 同行号 / 待定）
3. 进入选题池
4. 标记优先级
5. 生成结构草稿
6. 进入写作或等待
7. 发布后归档
8. 复盘转入记忆

## 二、状态定义

- inbox：刚进入系统，尚未判断
- planned：已确认纳入选题池
- drafting：正在写
- ready：待发布
- published：已发布
- paused：暂缓
- dropped：废弃

## 三、每周维护动作

1. 清理新增想法
2. 为双号各补充至少 3 个可写选题
3. 更新本周发布计划
4. 标记已完成 / 暂缓事项
5. 复盘表现与后续可复用点

## 四、程序维护目标

脚本至少支持：
- 查看当前选题池
- 按账号筛选
- 按状态筛选
- 新增选题
- 输出本周任务面板

## 五、归档要求

- 战略规则：docs/
- 数据表：data/
- 模板：templates/
- 可执行脚本：scripts/
- 后续内容草稿：drafts/（后续可增设）
