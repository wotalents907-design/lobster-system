# 备忘录｜2026-04-04 多代理一致性与抗失忆升级

## 这次改了什么

我先把“多代理一致性”和“换模型/重启不失忆”这件事，从讨论变成了本地项目文档。

本次已产出：
1. 多代理统一规则清单 v1
2. 结构化记忆 / 轻图谱 schema v1
3. 本次升级项目 README

## 为什么要改

因为当前多代理体系天然会遇到几个问题：
- 同一命令，不同代理反应不一致
- 换会话、换模型后容易失忆
- 长期任务状态容易断档
- 重要决策和规则难复用
- 协作容易靠聊天凑，而不是靠系统稳住

## 改完现在能做到什么

目前先做到的是“把规则立起来”：
- 明确哪些必须统一，哪些可以保留差异
- 明确内部升级默认直接做，不再反复请示
- 明确关键记忆必须外置，不依赖模型上下文
- 明确任务状态要统一枚举
- 明确本地是规则和记忆的主版本，GitHub 只是辅助沉淀

换句话说：
现在已经有了后续持续升级的骨架，不再只是口头共识。

## 目前还没做到的

还没有正式把现有记忆批量迁移到 `memory/graph/` 结构化层；
也还没把这些统一规则自动注入到所有代理各自的角色文件中。

## 下一步准备干什么

下一步建议继续做：
1. 建立 `memory/graph/` 目录和初版文件
2. 先迁移最值钱的 4 类信息：
   - 用户偏好
   - 活跃任务
   - 已拍板决策
   - 长期规则
3. 再考虑把统一规则同步进各代理启动文件/角色文件

## 一句话总结

这次不是“增加了一个新插件”，而是开始把整个代理体系，往“制度化、持久化、抗失忆”的方向改。
