# 多代理一致性与抗失忆升级项目

## 目标

把当前代理体系从“主要靠聊天上下文驱动”升级为“规则驱动 + 外置记忆驱动”，降低以下问题：

- 换模型后行为漂移
- 系统重启或开新会话后失忆
- 多代理对同一指令反应不一致
- 长期任务状态断裂
- 决策、规则、经验难以复用

## 本项目当前产物

- `UNIFIED_RULES_V1.md`：多代理统一规则清单 v1
- `GRAPH_MEMORY_SCHEMA_V1.md`：结构化记忆/轻图谱初版 schema
- `UPGRADE_MEMO_2026-04-04.md`：本次升级的人话备忘录

## 设计原则

1. 关键记忆必须外置，不依赖模型上下文
2. 本地规则为主，GitHub 只做抽象沉淀/同步备份
3. 统一规则、记忆、状态、职责
4. 保留风格、视角、专长差异
5. 先轻量化落地，再逐步自动化

## 下一步建议

1. 把统一规则同步进相关代理的启动/角色文件
2. 建立 `memory/graph/` 持久化目录
3. 先迁移：用户偏好 / 活跃任务 / 已拍板决策 / 长期规则
4. 后续再接入自动抽取与召回
