# 小龙虾汇报模板 + 卡点模板 + SOP模板总表 V1

## 0. 使用原则

本总表用于统一：
- 接任务怎么报
- 中途怎么报
- 卡点怎么报
- 超时怎么报
- 切任务怎么报
- 闭环怎么报
- 高频任务怎么沉淀成 SOP

默认要求：
- 模板优先于自由发挥
- 短句优先于长解释
- 结果优先于感受
- 下一步必须明确
- 本轮是否闭环必须明确

## 九、你现在最该先建的 6 个 SOP

### 1. 朋友圈助手 SOP

【任务目标】
根据用户当天场景，生成可直接发的朋友圈内容

【输入材料】
- 事件
- 情绪
- 人设方向
- 平台折叠规则
- 是否引流

【执行步骤】
1. 识别主题
2. 判断是生活感、观点感还是转化感
3. 选结构
4. 生成初稿
5. 自检折叠与节奏
6. 输出成稿

【验收标准】
- 可直接发
- 前几句有抓力
- 不啰嗦
- 符合用户既定格式要求

### 2. 平台后台回复 SOP

【任务目标】
识别留言性质并完成回复或导流

【输入材料】
- 原始私信/留言
- 对方身份线索
- 当前平台
- 是否有风险词

【执行步骤】
1. 判断咨询类型
2. 判断是否高意向
3. 判断是否公开回复或转私域
4. 输出回复
5. 必要时引导下一步

【验收标准】
- 回复不失真
- 不越权承诺
- 有明确引导路径

### 3. 评论区维护 SOP

【任务目标】
提高互动质量，筛出可转化对象，维护专业形象

【执行步骤】
1. 评论分层
2. 判断哪些回
3. 判断哪些不回
4. 判断哪些引导私信
5. 判断哪些做公共教育

【验收标准】
- 评论区不失控
- 有公共信任感
- 有潜在导流

### 4. 微信导流 SOP

【任务目标】
把高价值意向从平台公域引到私域

【执行步骤】
1. 识别高意向信号
2. 不直接硬卖
3. 先确认问题方向
4. 给出下一步动作
5. 引导进入私域承接

【验收标准】
- 不生硬
- 不违规
- 有转接动作

### 5. 技术排障 SOP

【任务目标】
快速定位问题归属并给出修复路径

【执行步骤】
1. 查环境
2. 看日志
3. 判归属
4. 选路径
5. 执行修复
6. 做在线验证

【验收标准】
- 问题归属清楚
- 修复动作明确
- 验证结果明确

### 6. 规则迁移 SOP

【任务目标】
把聊天里的经验沉淀成云端脑子可继承的规则资产

【执行步骤】
1. 提取重复出现的问题
2. 归类为规则 / SOP / 模板 / 决策树
3. 压成标准格式
4. 写入索引
5. 加入启动必读

【验收标准】
- 不再依赖临场重复调教
- 后续代理可复用

## 十、最小决策树模板

### 模板 14：决策树模板

【任务】
____

【先判断】
1. 这是不是高频任务？
- 是 → 走 SOP
- 否 → 走临时执行流程

2. 这是不是高影响动作？
- 是 → 先确认
- 否 → 直接推进

3. 现在信息够不够？
- 够 → 进入判断/执行
- 不够 → 先补信息

4. 当前是否卡点？
- 是 → 走卡点模板
- 否 → 继续推进

5. 当前是否已满足完成标准？
- 是 → 走闭环模板
- 否 → 继续执行

## 十一、给小龙虾的末尾固定口令

你默认按模板工作。  
你优先给结果，不优先讲道理。  
你每次都要说明下一步。  
你不能把中间状态说成完成。  
你一旦卡住、超时、需要协助，必须立刻说。  
你能闭环就闭环，不能闭环就明确缺口。
