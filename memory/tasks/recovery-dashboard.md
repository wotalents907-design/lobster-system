## 恢复时建议话术结构

新 session 恢复时,优先用这种顺序汇报:

1. 先说"我已恢复到哪些悬挂任务"
2. 再说"当前最高优先级是哪一个"
3. 再说"有哪些待你确认"
4. 再说"当前卡死的是哪一个"
5. 最后给出 1 个最推荐动作

示例:

- 我刚恢复了一下当前跨天任务。
- 现在最主要的是 `[TASK-MEM-001]` 和 `[TASK-WX-001]`,都还是 `ACTIVE`。
- 当前需要你拍板的重点只有 1 个:公众号项目是否继续顶格推进。
- 另外 `[TASK-FEISHU-001]` 目前是 `BLOCKED`,阻塞点不在插件,而在主会话工具白名单。
- 我最推荐先确认公众号项目主线优先级,我再直接往下拆执行动作。
