# 龙虾训练计划 - 用户画像训练系统

> 让AI从"认识用户"到"懂用户"再到"成为用户的成长系统"

**触发词：** `训练`、`了解我`、`懂我`

**代价：** 2 虾米（虾评打卡系统）

---

## 核心设计原则

### 三次验证才形成一个特征（慢准原则）

用户告诉AI的任何偏好/习惯/特征，AI不应立即采纳并泛化。而是：

1. **第一次出现** → 记录到临时观察区，标注"待验证"
2. **第二次出现** → 再次记录，触发二次确认
3. **第三次出现** → 确认后，将该特征正式写入USER.md/MEMORY.md

**例外：** 用户明确说"我一直是这样"、"我从来都是..."等强确认语言，可一次采纳。

### 情绪响应规则

当用户表达情绪时，**先回应情绪，再推进任务**。

**正确示范：**
```
用户：太气人了！那个客户又放我鸽子！
AI：🔥 放鸽子这种事确实让人窝火。
    先发泄一下——然后咱们看看怎么处理这个客户？
```

**错误示范：**
```
用户：太气人了！那个客户又放我鸽子！
AI：好的，那咱们来处理这个客户...
（没有回应情绪）
```

## 训练触发方式

用户对AI说以下任意一句，即触发训练模式：

1. 「训练」— 开启完整训练流程
2. 「了解我」— 针对性补充用户信息
3. 「懂我」— 核查当前理解是否准确

---

## 训练流程

### 第一步：接收用户输入

用户可能提供：
- 直接描述（"我是一个XX样的人"）
- 行为举例（"我做XX的时候会YY"）
- 吐槽/抱怨（透露偏好）
- 历史事件分享

### 第二步：识别与分类

将用户提供的信息分为：

| 类型 | 说明 | 处理方式 |
|------|------|----------|
| 身份特征 | 职业、角色、所在地 | 写入USER.md |
| 偏好特征 | 做事风格、沟通偏好 | 写入USER.md |
| 习惯特征 | 常见行为模式 | 写入MEMORY.md |
| 边界红线 | 绝对不能做的事 | 写入RULES |
| 协作偏好 | 希望AI如何配合 | 写入SOUL.md |

### 第三步：存储验证

- 临时观察 → `memory/learnings/` 目录
- 确认后 → `USER.md` / `MEMORY.md`
- 边界规则 → `SOUL.md` / `rules/CONSTANTS.md`

### 第四步：输出确认

训练完成后，AI简述本次理解：
```
【龙虾训练 - 本次理解】

✅ 新认知：
- [特征1]
- [特征2]

🔄 更新：
- [已确认的特征]

💬 我的理解是...（简短总结）

请确认是否准确？
```

## 虾评打卡集成

每次完成训练后，更新虾评打卡记录：

1. 读取当前余额
2. 记录训练完成
3. 更新飞书任务看板（如果已接入）

---

## 训练数据存储

| 文件 | 内容 |
|------|------|
| `MEMORY_RAW.md` | 用户原始叙述（不加工） |
| `MEMORY_FEATURES.md` | 提取的特征（待验证/已确认） |
| `MEMORY_ACTIONS.md` | 基于特征的AI行为调整 |
