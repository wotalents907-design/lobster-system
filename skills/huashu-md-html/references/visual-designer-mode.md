# 视觉艺术传达设计师模式

> 兜底脚本帮你「机械套版」，设计师模式帮你「为内容定制视觉表达」。
> 这是能力 2（md → html）的进阶用法——不消耗 token 的兜底之外的另一条路径。

---

## 两条路径并存

| 模式 | 是否耗 token | 适合场景 | 工作流 |
|------|---|---|---|
| **兜底模式** | ❌ 不耗 | 已知内容类型 + 想要快、不挑细节 | `scripts/md_to_html.py --theme xxx` 一条命令 |
| **设计师模式** | ✅ 耗 | 让 AI 读懂内容、推荐方向、定制视觉表达 | AI 介入 4 步工作流 |

两个模式不冲突。**「视觉艺术传达设计师」是兜底模式之上的可选升级层**——
你大部分时候用兜底，遇到「这篇内容值得做得更精彩」时切换到设计师模式。

---

## 设计师模式的 4 步工作流

借鉴 huashu-design 的 Junior Designer + Fallback 模式，分 4 步——

```
md 输入
  ↓ ① 阅读：AI 读 md（title / 章节结构 / 内容调性 / 受众线索）
  ↓ ② 推荐：3 个差异化设计方向，每个来自不同流派 + 代表参考
  ↓ ③ 用户拍板：选一个方向
  ↓ ④ 实现：基于该方向 → 在 4 主题之一上定制（CSS 变量 / inline 元素 / 版式细节）
自包含 HTML 单文件
  ↓ 可选 ⑤ 5 维度自评：哲学一致性 / 视觉层级 / 细节执行 / 功能性 / 创新性
```

---

## 第 1 步 · 阅读 md 找设计线索

读 md 的时候，AI 应该看出这几个维度：

| 维度 | 怎么看 | 影响什么 |
|------|--------|----------|
| **内容类型** | essay？产品介绍？技术报告？教程？小说？ | 决定 baseline 主题 |
| **受众身份** | 开发者？普通读者？投资人？编辑？ | 决定字号 / 严谨度 / 留白量 |
| **调性气质** | 严肃克制？热情活泼？技术冷静？文艺？ | 决定字体 / 配色 / 视觉节奏 |
| **结构密度** | 信息密度高（多表格）？还是散文型？ | 决定布局（宽体 / 单栏） |
| **品牌锚点** | 是否有已有视觉系统（如花叔橙皮书橙）？ | 决定能不能复用已有色板 |

**反例**：「这是一篇 markdown，转 html」——什么都没说，套兜底就行。
**正例**：「这是一个开源 skill 的产品介绍 README，受众是开发者 + AI 创作者，调性是「暖色科技」，需要让人愿意装它」——足够推荐 3 个方向。

---

## 第 2 步 · 推荐 3 个差异化方向

按 huashu-design 的方法——**每个方向来自不同流派**，避免「同一种风格的三种微调」。

### 设计流派池（来自 huashu-design 的 5 流派 × 20 哲学）

可以从下面挑：

| 流派 | 代表 | 适合 |
|------|------|------|
| **Pentagram 信息建筑** | Paula Scher / Pentagram NY | 严肃产品、企业落地页、信息密集 |
| **Tufte 信息可视化** | Edward Tufte | essay、学术、数据驱动 |
| **Kenya Hara 东方极简** | 原研哉 / MUJI | 文艺、产品哲学、克制 |
| **Anthropic 暖色科技** | Anthropic / Notion 早期 | 开发者工具、温暖品牌 |
| **Stripe Docs** | Stripe / Linear | 技术文档、SaaS 工具 |
| **Penguin Books** | Penguin / 出版业 | 书籍、长内容、阅读优先 |
| **Sagmeister 实验先锋** | Stefan Sagmeister | 创意、艺术、张扬 |
| **Field.io 运动诗学** | Field.io / 数字艺术 | 实验、抽象、新媒体 |
| **Bauhaus 几何包豪斯** | Bauhaus / Vignelli | 严谨、几何、教科书 |

**每个方向至少包含**：
- 哲学锚点（一句话定调）
- 配色（3-4 个具体色值）
- 字体组合（中英文 + 代码）
- 关键视觉锚点（hero 怎么做、表格怎么做、卡片怎么做）
- 适合什么场景

**例子**（README 的设计推荐）：

> **方向 A · Pentagram 产品介绍页**
> 视觉哲学：信息建筑——大字号 sans + 12 列网格 + 严格分级 + 一抹强调色
> 配色：象牙白 #FAFAF7 + 墨黑 #1A1A1A + 一抹 #C2410C 橙
> 锚点：满版 Hero、4 列横向卡片、Tufte 风纤细分隔线
>
> **方向 B · Kenya Hara 极简白书**
> 视觉哲学：负空间 + 衬线 + 一处 120% 细节签名
> 配色：纯白 + 灰阶 + 一抹赤陶橙仅用于关键链接
> 锚点：克制字号、大量留白、Tufte 风纤细分隔
>
> **方向 C · Anthropic 暖色科技**
> 视觉哲学：奶油底 + 赤陶橙 + 黑线条 + serif title
> 配色：奶油 #FDF8EE + 赤陶橙 #C2410C + 墨黑 + 灰阶
> 锚点：Hero 装饰图、2×2 网格卡片、暖色 callout

3 个方向给完之后，**告诉用户你推荐哪个 + 为什么**，但让用户拍板。

---

## 第 3 步 · 用户拍板

不要默认替用户选。设计判断有主观成分，让人来决定。**这一步是设计师模式区别于兜底模式的关键**——
兜底是「机械给一个」，设计师是「给选择 + 告诉你各自适合谁」。

---

## 第 4 步 · 实现

确定方向之后，**不要直接用 pandoc + 现成主题**——pandoc 出来的是「机械版」。设计师模式要做的事：

1. **不依赖 pandoc**——直接手写自包含 HTML，全 inline CSS。这样定制空间最大。
2. **以 md 内容为蓝本**——按 md 的章节结构组织 HTML 区块，但**每个区块可以定制版式**（卡片 / 表格 / 代码块 / 装饰）。
3. **应用方向的视觉语言**——配色、字体、间距、字号、装饰、动效（如果需要）都按方向走。
4. **保持反 AI slop 底线**——参见 `references/anti-ai-slop.md`，紫渐变、emoji 图标、SVG 画人物一律不要。
5. **图片用相对路径**——README 里引用的图（如 `demos/v3-hara-hero.gif`、`examples/output/article.png`）保持原样，不要 base64 嵌入（防止文件巨大）。
6. **响应式必做**——`@media (max-width: 720px)` 至少处理一道，移动端不能瘫。

---

## 第 5 步（可选）· 5 维度自评

参考 huashu-design 的评审清单，每个维度 0-10 分——

| 维度 | 什么算 10 分 |
|------|--------------|
| **哲学一致性** | 整个页面里没有违背所选流派的元素（如选了 Kenya Hara 但用了赛博渐变） |
| **视觉层级** | 扫一眼能立刻分清「这是 hero / 这是表格 / 这是关键 CTA」 |
| **细节执行** | 字距、行高、padding、border-radius 都过了一遍，没有「编辑器默认值」感 |
| **功能性** | 信息可读、操作清楚、链接都通 |
| **创新性** | 有一处「120% 细节签名」让人愿意截图保存 |

总分 40+ 才算交付级别。低于 35 的回去改。

## 兜底 vs 设计师 · 怎么选

| 你说的话 | 我应该用什么模式 |
|---|---|
| 「把 md 转 html」 | 兜底（`--theme article` 默认） |
| 「用 article / report 主题做一下」 | 兜底（直接 `--theme xxx`） |
| 「给这个 md 做个出色的 html」 | **设计师模式**（推荐 3 个方向） |
| 「我要这个文章的 html 版」 | 反问：是要快速渲染（兜底）还是要定制设计（设计师）？ |
| 「让我看看几种风格」 | **设计师模式**（必给 3 个差异化方向） |
| 「按 Anthropic 风格做」 | **设计师模式**（流派已定，跳过推荐直接做） |

---

## 一句话总结

**兜底是「我给你什么」，设计师是「你的内容应该被怎么表达」。**
后者更贵（耗 token），但产出有出版品味——
让 README 看起来像个真产品的官网，让 essay 看起来像 The New Yorker 的专栏，让技术报告看起来像 Stripe 的 docs。

这就是「视觉艺术传达设计师」的工作。
